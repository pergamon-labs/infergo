# InferGo v0.2.0-alpha.3

InferGo `v0.2.0-alpha.3` is the current hardened alpha baseline for internal
and early external evaluation.

## Summary

This prerelease keeps the same narrow public alpha shape:

- library-first Go inference for backend services
- family-1 BYOM export/import for supported encoder-style text-classification
  models
- parity-backed validation against source-model references
- optional HTTP serving for exported bundles

It does **not** claim broad model compatibility.

## Highlights since alpha.2

- stronger family-1 tokenizer-boundary enforcement for exported bundles
- validated RoBERTa-style ByteLevel BPE tokenizer-json support for raw-text
  family-1 bundles
- stable byte and character offsets for curated raw-text NER flows
- exporter UX hardening for first-time users
- documented maintainer benchmark snapshot and regression workflow
- enforced alpha native bundle compatibility for `infergo-native` `1.0`
  bundles

## Supported alpha path

The main alpha path is:

1. export a supported family-1 model into an InferGo-native bundle
2. load that bundle in Go without Python at runtime
3. call it directly or expose it over HTTP
4. verify parity against the source reference

## Notes

- the public alpha native bundle contract is currently pinned to
  `bundle_format=infergo-native` and `bundle_version=1.0`
- `torchscript` remains an experimental, optional compatibility bridge and
  depends on `libtorch`
- token classification remains a curated-pack/sample-service alpha story, not a
  second public BYOM family
- ONNX remains a later design question, not part of the current alpha contract
