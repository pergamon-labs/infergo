# InferGo v0.2.0-alpha.2

InferGo `v0.2.0-alpha.2` is the first cleaned public alpha line after removing
private internal entity-resolution bridge material from the public repository.

## Summary

This alpha is intentionally narrow.

It is centered on:

- library-first Go inference for backend services
- family-1 BYOM export/import for supported encoder-style text-classification
  models
- parity-backed validation against source-model references
- optional HTTP serving for exported bundles

It does **not** claim broad model compatibility.

## Highlights

- installable `infergo-export`, `infergo-serve`, and `infergo-parity` CLIs
- public-safe family-1 export -> load -> serve -> parity walkthrough
- raw-text and paired-text serving for supported exported family-1 bundles
- stable public Go API examples for curated packs and exported bundles
- NER validated in a public-safe sample Go service
- public docs simplified around a library-first alpha story

## Supported alpha path

The main alpha path is:

1. export a supported family-1 model into an InferGo-native bundle
2. load that bundle in Go without Python at runtime
3. call it directly or expose it over HTTP
4. verify parity against the source reference

## Notes

- `torchscript` remains an experimental, optional compatibility bridge and
  depends on `libtorch`
- token classification remains a curated-pack/sample-service alpha story, not a
  second public BYOM family
- ONNX remains a later design question, not part of the current alpha contract
