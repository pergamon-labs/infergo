# InferGo v0.2.0-alpha.1

InferGo is a Go-native inference toolkit for backend services.

This alpha release makes the project usable as a narrow bring-your-own-model
toolkit for one documented public family, while keeping the support contract
explicit.

## What this alpha is

InferGo alpha is for teams that want to:

- export a supported model once
- load it in Go without Python at runtime
- use it directly in an existing Go service
- optionally expose it over HTTP
- validate parity against a reference implementation

## What is included

### Public family-1 path

- installable `infergo-export`
- installable `infergo-serve`
- installable `infergo-parity`
- documented family-1 export/import workflow
- exported bundle loading through `infer.LoadTextBundle(...)`
- tokenizer-backed raw-text support for the current supported tokenizer subset
- paired-text export and prediction path
- parity checks against generated source references

### Curated validation paths

- curated native text-classification packs
- curated native token-classification packs
- sample NER extraction service in Go

### Internal dogfood path

- experimental internal-first family-2 `entres` bridge
- local load / serve / parity path for the current entity-resolution models

## What is not included

InferGo alpha does **not** mean:

- arbitrary Hugging Face runtime loading
- arbitrary `.pt` loading
- token-classification BYOM export/import
- general transformer execution in Go
- ONNX runtime support
- training or fine-tuning
- gRPC serving

## Recommended getting-started path

1. Install the tools:

```bash
go install github.com/pergamon-labs/infergo/cmd/infergo-export@latest
go install github.com/pergamon-labs/infergo/cmd/infergo-serve@latest
go install github.com/pergamon-labs/infergo/cmd/infergo-parity@latest
```

2. Follow [`docs/GETTING_STARTED.md`](../GETTING_STARTED.md)

3. For the full public alpha path, follow
   [`docs/alpha-family-1-walkthrough.md`](../alpha-family-1-walkthrough.md)

## Known limitations

- family-1 raw-text support is intentionally limited to the current validated
  tokenizer subset
- token classification is still a curated-pack/sample-service alpha story
- family 2 is still internal-first and experimental
- HTTP is optional and secondary to the embedded Go library path

## Verification snapshot

Current alpha-hardening verification already includes:

- installed `infergo-export`, `infergo-serve`, and `infergo-parity`
- clean-room export of
  `distilbert/distilbert-base-uncased-finetuned-sst-2-english`
- parity pass against the generated source reference
- bundle load from a tiny Go app outside the repo
- standalone HTTP serving and `/predict` success against that exported bundle

Current benchmark snapshot is documented in [`BENCHMARKS.md`](../../BENCHMARKS.md).

## Who this release is for

This release is a good fit for:

- Go backend teams who want in-process inference
- teams replacing Python runtime dependencies in narrow serving paths
- engineers willing to work within an explicit supported family boundary

This release is not a good fit yet for:

- teams expecting broad model-format compatibility
- teams needing platform-scale serving features
- teams expecting training or general ML framework capabilities
